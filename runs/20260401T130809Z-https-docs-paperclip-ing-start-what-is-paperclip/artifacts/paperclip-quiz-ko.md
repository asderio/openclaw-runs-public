# Paperclip 퀴즈

## 문제 1
Paperclip 아키텍처에서 `Control Plane`과 `Execution Services`의 가장 중요한 차이는 무엇인가?

- [ ] Control Plane이 실제 LLM 추론을 담당하고, Execution Services가 데이터베이스와 인증을 담당한다.
- [x] Control Plane은 오케스트레이션을 담당하는 중앙 신경망이고, Execution Services는 외부 런타임과 연결되는 브리지다.
- [ ] Execution Services가 조직 계층을 정의하고, Control Plane은 토큰 소비만 모니터링한다.
- [ ] Control Plane은 분산된 에이전트 네트워크이고, Execution Services는 단일 명령 체계를 제공한다.

힌트: 어떤 계층이 "두뇌" 역할을 하고, 어떤 계층이 외부 실행 환경과 연결되는지 떠올리면 된다.

## 문제 2
여러 AI 에이전트가 같은 작업을 중복 수행하지 않도록 Paperclip이 사용하는 메커니즘은 무엇인가?

- [ ] 가장 빠른 에이전트가 웹훅으로 먼저 가져가는 방송형 선점 방식
- [x] `Atomic Checkout`을 통해 이미 점유된 작업에 접근하면 `409 Conflict`를 발생시키는 방식
- [ ] 하드웨어 ID 기준으로 작업을 지역 분산하는 방식
- [ ] 사람이 모든 작업 할당을 실시간 승인하는 `Board Override` 방식

힌트: 데이터베이스에서 "쪼갤 수 없는 단일 연산"을 뜻하는 용어를 떠올리면 된다.

## 문제 3
Paperclip에서 `Heartbeats`는 어떤 역할을 하는가?

- [ ] LLM 연결이 끊기지 않도록 24시간 유지되는 상시 데이터 스트림
- [ ] PostgreSQL이 무거운 쿼리를 수행할 때 쓰는 CPU 사이클 단위
- [x] 에이전트가 잠깐 깨어나 작업을 수행하고 결과를 보고하는 짧은 실행 창
- [ ] 에이전트의 상태가 위험 수준일 때 강제로 종료하는 킬 스위치

힌트: 상시 실행이 아니라 필요할 때만 깨어나는 리듬이다.

## 문제 4
Paperclip의 비용 통제 모델을 가장 잘 설명하는 것은 무엇인가?

- [ ] 모든 에이전트가 공유 기업 카드에 무제한 접근한다.
- [ ] 비용은 Company 수준에서만 관리되고 에이전트는 자율적으로 조정한다.
- [x] 각 에이전트에 센트 단위의 월별 `token salary budget`을 배정해 burn rate를 통제한다.
- [ ] 과거 성과와 ROI에 따라 예산이 자동으로 계속 재조정된다.

힌트: Paperclip은 에이전트를 직원처럼 다루며, 비용도 직원별로 관리한다.

## 문제 5
다음 중 사람 이사회(Board)의 개입이 명시적으로 필요한 것은 무엇인가?

- [ ] 낮은 우선순위 작업의 atomic checkout
- [ ] 하트비트 트리거 전송
- [x] CEO 전략 승인과 하위 에이전트 채용
- [ ] 모노레포 로그를 프론트엔드에 실시간 번역하는 일

힌트: 조직 리스크가 큰 행동일수록 인간 승인 게이트가 붙는다.

## 문제 6
엄격한 트리 계층 조직 구조가 만드는 운영 리스크는 무엇인가?

- [ ] 보고 체계가 불명확해서 작업 출처를 추적할 수 없다.
- [x] 특정 관리자 에이전트가 병목이 될 수 있고, 매트릭스형 협업에 약하다.
- [ ] 에이전트가 예산이 낮다고 노조를 결성한다.
- [ ] 계층 구조가 재귀 루프를 만들어 비용이 기하급수적으로 늘어난다.

힌트: 지나치게 수직적인 조직은 유연성을 해칠 수 있다.

## 문제 7
Execution Plane의 에이전트들은 어떻게 Control Plane에 결과를 보고하는가?

- [ ] Control Plane이 에이전트 메모리에 직접 명령을 주입한다.
- [x] 에이전트가 API를 호출해 `phone home` 방식으로 상태와 결과를 보고한다.
- [ ] Control Plane에서 Adapter로만 단방향 명령이 내려가고 피드백은 없다.
- [ ] 물리적 시리얼 케이블로만 연결된다.

힌트: 본사에 전화로 보고하는 비유가 문서에 나온다.

## 문제 8
Paperclip 모노레포를 로컬에서 실행하기 위한 핵심 기술 전제는 무엇인가?

- [ ] 80GB VRAM 이상의 GPU 클러스터
- [x] Node.js 20+, pnpm 9, 그리고 PostgreSQL 기반 환경
- [ ] Board 전용 하드웨어 콘솔
- [ ] Python 3.12와 LangChain, CrewAI 사전 설치

힌트: 현대적인 TypeScript 웹 애플리케이션 실행 환경이다.

## 문제 9
왜 Paperclip은 Jira 같은 작업 관리 도구가 아니라 `Control Plane`으로 설명되는가?

- [ ] Jira는 AI 에이전트용이고 Paperclip은 인간 협업용이라서
- [x] Paperclip은 전체 자율 AI 조직의 구조, 예산, 거버넌스, 실행 제약까지 관리하기 때문에
- [ ] Paperclip이 영어 목표를 기계어로 컴파일하기 때문에
- [ ] Paperclip이 블록체인 기반 프로토콜이기 때문에

힌트: 일감을 적는 것과 조직을 운영하는 것은 다르다.

## 문제 10
인간 이사회(Board) 거버넌스 모델의 전략적 한계로 가장 적절한 것은 무엇인가?

- [ ] Board는 하트비트가 시작되면 더 이상 개입할 수 없다.
- [x] 인간 승인 게이트가 AI 조직의 속도와 확장성을 늦출 수 있다.
- [ ] Board도 AI 에이전트로만 구성되어 순환 논리에 빠진다.
- [ ] Board 결정은 CEO 에이전트가 무시할 수 있다.

힌트: 안전성과 속도는 보통 상충한다.
